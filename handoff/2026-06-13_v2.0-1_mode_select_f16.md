## 設計哲學
- 玩法多元並存：`free / dogfight / race` 三模式可選；空戰下再分子模式（氣球靶 / PvP / 敵機）。
- tone ladder：F-16 是擬真端機種（戰鬥），voxel 視覺仍玩具。

## 硬性約束
- 模式框架**不重建**：擴 `hud-slots.js` 的 `HUD_MODES`（加 `dogfight`/`race` 與其 eligible slots）、`context-keys.js` 的 key config（加 `DOGFIGHT_KEYS`/`RACE_KEYS`，本階段先佔位、內容由 v2.0-5/v2.1-1 接）。
- F-16 **飛行參數化**：機型參數（V_MAX/阻力/轉彎/油箱…）從 hardcoded 抽成 per-plane spec（建議進 `airspace`/`plane spec`），T-34C 走同一條路、數值不變。
- 不動連線協定（btn bitmask 的開火位元由 v2.0-2 加）。

## 分 Phase 實作
**P1 — 模式狀態 + 選單 UI**：`gameMode: free|dogfight|race`；空戰子模式 `dogfightMode: balloons|pvp|ai_1v1|ai_2v2`。模式選單（玩具櫃 UI，填 CenterSlot/獨立 overlay）。
**P2 — HUD/keys 模式 config**：擴 `HUD_MODES`（dogfight 亮 武器/彈藥/鎖定/計分槽位佔位、race 亮 名次/計時佔位）；context-keys 加 `DOGFIGHT_KEYS`/`RACE_KEYS` 佔位。
**P3 — F-16 機型**：`models/f16.js` voxel（紅白→塗裝由 slot 著色）；飛行參數抽成 plane spec（F-16 快/靈活/耗油兇）；解鎖**沿用 v1.2 機制**（時數＋里程碑），v1.2 未到則暫設可選。
**P4 — 測試**：vitest（模式切換無串、HUD_MODES 映射、F-16 vs T-34C 參數載入、flight-model 多機型回歸）；e2e（選單切 free↔dogfight↔race、選 F-16 起飛）。
