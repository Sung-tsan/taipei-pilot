## 設計哲學
- 安全模式開關＝**後果嚴重度旋鈕**（安全→溫和→真實），與 ❤️ 上限旋鈕同一條軸（沿用小司機 #13 家長旋鈕哲學）。
- 一個旋鈕管多件事：之後 V3 天氣嚴重度也吃這檔（本輪先留 enum，天氣 V3 接）。

## 硬性約束（拍定，照 ROADMAP §4 v1.1 設計細則）
- 三檔後果分支：**安全＝彈開、不墜、∞ 重試**；**溫和＝❤️ 計數（可調上限 1–5/∞），歸零回機場**；**真實＝漸進 damage：完好→冒煙受損→再撞墜毀**。
- 後果軸**預設安全**；切換**放設定頁**；真實模式**不需二次確認**。
- 三檔機制各自獨立（彈開 / ❤️ / 漸進 damage 不混用）。

## 既有資產照抄

- 設定 overlay／旋鈕：v1.1-0 的設計語言（Tactile 按壓）+ StatusSlot。

## 分 Phase 實作
**P1 — 後果軸 state**：`consequenceMode: 'safe'|'gentle'|'real'`（display 端狀態，per session 從 localStorage 載）；參數化 seam 預留 `airspace.consequenceDefaults`（本輪用全域預設）。

**P2 — 撞擊後果分支**（接 `collision.js`）：
- safe → 既有彈開＋回饋。
- gentle → ❤️−1（撞/壓太低共用計數）；歸零→「回機場休息」回跑道、❤️ 補滿。
- real → **damage state machine**：`intact → smoking`（首次撞/劣質迫降：冒煙視覺＋警告）→ `destroyed`（再撞：墜毀→回跑道）。`smoking` 是否微降性能＝**本輪實作為「僅視覺＋警告、不降性能」**（降性能留 backlog，避免疊太多變數）。

**P3 — ❤️ 計數 + 上限旋鈕 + 設定頁**：
- ❤️ 顆數渲染進 `StatusSlot`（顆數跟上限走）。
- 設定頁（gear 入口）：後果模式三選一 + ❤️ 上限（1–5/∞，僅 gentle 有效）。Tactile 旋鈕。

**P4 — localStorage**：`tp_consequence_mode`（預設 `safe`）、`tp_mishap_limit`（預設 3）。換手機掃碼不掉（狀態在電腦端，沿用 spec §7）。

**P5 — 測試**：
- vitest：三檔分支判定（撞擊→各檔結果）、damage state machine（intact→smoking→destroyed）、❤️ 計數與上限、歸零回機場、localStorage 讀寫。
- e2e：設定頁切換三檔 + ❤️ 上限調整生效。
