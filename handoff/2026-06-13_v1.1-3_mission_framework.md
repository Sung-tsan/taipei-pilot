## 分 Phase 實作
**P1 — 任務 data model + airspace seam**：
- `mission = { id, type, targetId|rings[]|altRule, prompt /*DRAFT*/, order }`。
- 四型 enum：`landmark_find / ring_route / altitude / takeoff_landing`。
- 進 `airspace.taipei` 的 `missions[]` 與 `landmarks[].missions`。

**P2 — 四型判定邏輯**：
- 地標尋寶：飛進目標光圈（距離+高度容差）。
- 穿圈航線：依序穿過 rings（沿淡水河/基隆河折線生成圈）。
- 高度挑戰：維持/達到高度帶（「比 101 高」/「貼基隆河低飛」）。
- 起降練習：回松機落地 **或** 真實模式迫降成功（接 v1.1-2 事件）。

**P3 — 適應性挑選（純 JS）**：依距離/大小排序、失敗回填簡單、雙人各自佇列獨立；持續供題（達成→下一個）。

**P4 — 教育內容 DRAFT**：每地標一句 fact（≤20 字）+ 任務 prompt + 鳥瞰知識（河流走向/跑道 10-28 方位）；**全標 `DRAFT`**，集中一處便於 v1.1-5 校稿。

**P5 — 測試**：
- vitest：四型判定（光圈/rings 序/高度/起降）、適應性挑選排序與回填、雙人獨立供題、DRAFT 標記存在性檢查。
- e2e：自由飛中接到任務 → 達成 → 下一題（任務循環不卡）。
