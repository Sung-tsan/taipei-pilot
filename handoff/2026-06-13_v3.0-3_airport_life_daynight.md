## 設計哲學
- 直接解「停機坪前面沒飛機」抱怨——讓世界有人住的感覺。
- **靜態為主軸**（最便宜、最穩 perf）；移動體是 bonus，不擋版本。

## 硬性約束（拍定）
- **生活感清單（主軸，靜態）**：停機坪靜態飛機 / 跑道·滑行道燈 / 航廈窗光 / 風向袋 / 旋轉雷達 / 靜態地面車輛。
- **日夜＝開場可選固定時段**（白天 / 黃昏 / 夜），modulate `world.js` 光照；**純氛圍不懲罰**（夜不額外降能見度，除非同時起霧）。
- **移動車流 / 遠方環境機＝perf bonus**，看 draws 預算決定收不收，**不擋 DoD**。
- 靜態擺件一律 **merged / instanced geometry**（draws 紀律）。

## 既有資產照抄
- `src/display/scene/world.js`（光照/天空——日夜 modulate 光色/角度/環境光）。
- `src/display/scene/{airport,taipei,city-gen}.js`（擺件放置：停機坪/航廈/道路）。
- `src/voxel/build.js`（merged geometry；新擺件走同一條建模）。
- 風向袋：v3.0-2 已有風向邏輯，本階段加模型（同步擺動）。
- perf 量測：`?debug=1`（fps / draw calls 面板，既有）。

## 分 Phase 實作
**P1 — 靜態擺件**：停機坪飛機（重用 T-34C/F-16 voxel 當 prop）、跑道·滑行道燈、航廈窗光、旋轉雷達、靜態地面車輛。全 merged/instanced。**量 draws**。
**P2 — 風向袋模型**：接 v3.0-2 風向（指示功能）+ 擺件（隨風動）。
**P3 — 日夜**：開場可選白天/黃昏/夜；modulate world.js 光色/方向/環境光 + 夜間燈光亮（跑道/城市/航廈窗）；純氛圍。與 v3.0-1 天氣協調（夜+霧才降能見度）。
**P4 — 移動 bonus（看 perf）**：路上小車車流 / 遠方盤旋環境機；draws 不夠就砍，log 砍了什麼（不靜默）。
**P5 — 測試**：vitest（日夜光照狀態、擺件放置不壓地標/跑道、風向袋同步風向）；e2e（選夜→燈光亮；雙視口 perf 量測 assert draws < 300/視口）。
