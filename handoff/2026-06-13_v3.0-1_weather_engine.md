## 設計哲學
- 天氣＝參數化系統（B1）：`weatherProfile` 進 airspace，換空域＝換天氣機率，引擎零改動。
- **一個旋鈕管兩件事**：後果軸（安全/溫和/真實）同時當天氣嚴重度閘。

## 硬性約束（拍定）
- 天氣類型：晴 / 多雲 / 雨 / 霧（側風/亂流在 v3.0-2）+ 日夜（v3.0-3）。
- **觸發＝兩者都要**：平時照 `weatherProfile` 隨機 roll ＋ 天氣挑戰任務指定（任務在 v3.0-4）。
- **接後果軸**：安全＝永遠好天氣；溫和＝只溫和天氣；真實＝整張 profile 全開。
- **套用範圍＝台北（松山）**；`weatherProfile` schema 支援 9 機場、**數值標 DRAFT**；島嶼招牌天氣活化於 V5。

## 分 Phase 實作
**P1 — weather state + profile**：`weather = { type, ... }`；`airspace.weatherProfile`（每機場機率表，台北先給、其餘 DRAFT）。
**P2 — 隨機 roll + 後果閘**：進場依 profile roll（安全強制晴、溫和限溫和、真實全開）；天氣挑戰任務可覆寫（v3.0-4 接）。
**P3 — render（modulate world.js）**：霧＝fog 拉近+減亮；多雲＝雲密度/光；雨＝particle + 濕亮材質。雙視口共用同一 weather state。
**P4 — 測試**：vitest（profile roll 機率、後果閘三檔限制、weather state 切換、DRAFT 標記）；e2e（真實模式進場見霧/雨、安全模式恆晴）。
