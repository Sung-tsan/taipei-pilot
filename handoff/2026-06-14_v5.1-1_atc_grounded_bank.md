## 設計哲學
- **offline-LAN 鐵律不破**（server 純 relay、不連雲）→ ATC 生成走 **build-time 預生成 bank**，**runtime 不連雲**。
- 「生成」的好處（多樣不重複）在生成時拿，安全（預審）在烤進前把關。

## 硬性約束（拍定：預生成 bank，不連雲）
- **build-time 預生成**：用 Claude API（一次性 dev 工具，**建議 Haiku 便宜快**；實作引用 `claude-api` skill 確認 model/params）生一大批 grounded ATC phraseology 變體 → JSON bank。
- **三件套**（教訓 B4/B5）：① grounded prompt——只准用真實航管用語 + 該機場真實 data（呼號/跑道/gate/滑行道）② 結構驗證——產出過格式+關鍵詞 grounding 檢查，不合格丟棄 ③ **固定 clip 地板**——每階段一組固定 phraseology 保底 ④ runtime **逾時/缺變體 → fallback 固定地板**。
- **Sung 校審**：bank 標 DRAFT，Sung 審過才烤進遊戲（child-safety + 適齡 + 正確）。
- runtime：只從 bank 挑變體 + 無線電 TTS clip（band-pass+雜訊+咔，沿用 V4）；**不發任何雲端請求**。
- 中文簡化但保真實感（例：「小飛官 ABC，松山塔台，跑道 10，可以起飛了」）。

## 分 Phase 實作
**P1 — bank schema + 生成工具**：`scripts/gen-atc-bank.js`（build-time，呼叫 Claude API，grounded prompt 餵機場 data，產各階段 phraseology 變體）+ 結構/grounding 驗證（丟不合格）。
