## 1. 核心設計模型 — 兩條正交軸

這款不是「一種玩法」，而是一個 2D 設計空間。**任一玩法 × 任一後果＝一局有效遊戲。**

### 軸一・玩法模式（並存可選，不是二選一）
| 模式 | 內容 | 版本 |
|---|---|---|
| 合作探索 | 一起飛、找地標、完成任務 | V1 |
| 空戰 | co-op PvE（一起打）與 PvP（兄弟對戰）皆可選；非暴力鎖定（柔光/泡泡） | V2 |
| 競爭任務賽 | 誰先到地標／誰先穿完航線 | V2 |

### 軸二・後果模型（＝「安全模式」開關，後果嚴重度旋鈕）
| 檔位 | 撞擊／失誤後果 | 適用 |
|---|---|---|
| 安全（預設） | 彈開、不墜、∞ 重試 | 6 歲、第一次玩 |
| 溫和 | ❤️ 計數（可調上限 1–5/∞），歸零回機場 | 一般遊玩 |
| 真實 | 撞＝爆炸墜機；空戰被擊落＝出局；**可迫降機場外（水上/馬路）** | 大孩子／爸爸 |

> ❤️ 上限旋鈕與安全模式開關是**同一條後果軸**（沿用小司機 #13 家長旋鈕哲學）。
> **PvP design note**：spec 原本讓兩機幽靈穿透＝刻意避免「兄弟互撞變吵架」。做 PvP 必須靠後果軸接住——安全/溫和的 PvP「被打中＝冒煙暫退、幾秒重生」非永久淘汰；只有真實模式才「出局」；PvP 模式可關、預設給小小孩是合作。

## 2. 成長弧 — 「你解鎖的飛機，決定模擬的深度」

```
教練機（T-34C）          噴射機            民航機
  arcade ──────────────────────────────────► light sim
  自由飛/空戰/競速      更快/競速        塔台·滑行·登機·推機·航線
```

飛機從「玩具」一路長成「真的航空器」，遊戲也從 arcade 漸變成 light sim。這條弧是 V1→V5 的脊椎。

**Tone ladder（同一條弧的調性版）**：**玩具視覺恆定**（voxel + low-poly 共存，統一 flat-shaded＋色票，見 §11），但程序與音效的擬真度隨機種往上爬——教練機卡通（卡通飛彈、可愛聲）→ 噴射戰鬥機擬真（空對空/地飛彈、擬真爆炸）→ 民航機仿真（真實 ATC phraseology、滑行程序）。「調性跟著飛機長大」。

### V3 — 世界活起來（決策已鎖 2026-06-13）

**3A 天氣系統**
- 天氣類型：晴/多雲/雨/**霧**/**側風**/**亂流** + 日夜時段；進 airspace.json `weatherProfile`（每機場機率表）+ runtime weather state。
- **觸發＝兩者都要**：平時照 `weatherProfile` 隨機 roll（氣氛）＋ 明確「天氣挑戰任務」（如「霧中降落金門」）。
- **接後果軸（一個旋鈕管兩件事）**：安全＝永遠好天氣；溫和＝只溫和天氣；真實＝整張 profile 全開。
- **亂流/側風 vs 暈（鎖定）**：只在**真實模式**開、幅度很輕、配視覺風向指示（風向袋＋畫面風箭頭）、輕微磁吸幫接；**但磁吸有 threshold——超過則飛機受損或墜毀**（磁吸非無限寬容，天氣在真實模式有牙齒）。沿用 v0.2.4 減暈紀律。
- 招牌對應：霧→金門/馬祖南竿；側風→澎湖；亂流→花蓮/台東。

**3B 機場/城市生活感**（解「航廈前面沒飛機」抱怨）
- **靜態擺件＋燈光為主軸**：停機坪靜態飛機、跑道/滑行道燈、航廈窗光、夜景；河上船（已有）。
- 移動車流/遠方環境機＝看 perf 的 bonus，**不擋版本**。
- **日夜＝開場可選固定時段**（白天/黃昏/夜），不做流動循環（5 分鐘 session 看不到，留 stretch）。
- 「停機坪擺幾架靜態飛機」夠便宜，可當**快速 win 內容滴**提前到 v1.x。
- **風向袋＝一物兩用**（側風功能指示＋生活感擺件，橋接 3A/3B）。
- **perf spike**：雙視口 60fps、draw calls <300/視口；靜態擺件 merged/instanced，移動體節制。

## 6. 貫穿全程的三條軌

- **內容滴**（v1.1 後隨時可插，引擎零改動）：九機場分批滴出（高雄/台中 → … → 離島三場）、新地標、塗裝/航空公司。每滴一個就多一張機場、一種挑戰、一塊台灣地理。**到 V5 升級成航線目的地**。
- **音效軌**（跨版本長出，每版 handoff DoD 含「本版新音效」）：
  | 版本 | 新增音效 |
  |---|---|
  | v1（現況） | 螺旋槳引擎 loop（隨油門）、風聲、落地掌聲、喇叭、撞擊 |
  | v1.1 | 卡通爆炸、迫降依地形（水花/刮地/輪胎）、達成+點亮 success、❤️−1、慶祝煙火 |
  | v1.2 | 每機種不同引擎聲（噴射/ATR 渦輪/螺旋槳）、解鎖音 |
  | V2 | 發射聲、命中、被擊落爆炸、競速計時/名次 |
  | V3 | 風聲分級（側風更強）、雨聲、霧中悶音、亂流抖音、日夜環境底噪 |
  | V4 | 塔台聲（嗶+短語音）、地面車輛/pushback/登機廣播/滑行胎/空橋 |
  | V5 | 塔台語音生成（grounded TTS）、各機場環境底噪差異 |

  架構：①連續/反應式走參數化合成（引擎隨轉速、風隨側風），one-shot 走短取樣 clip（hybrid）②塔台 V4 短語音 clip → V5 生成 TTS（**仿真 phraseology**，grounded+逾時 fallback）③positional audio 留 polish ④**音效個性＝隨 tone ladder 分級，不是一律卡通**：卡通飛彈/低齡走可愛「噗咻」、空對空/空對地飛彈走擬真爆炸、塔台走仿真——視覺恆定（voxel + low-poly 共存，見 §11）、音效擬真度隨機種爬升；建議比照 DESIGN_CRAFT_GUIDE 立「音效個性指南」明訂這條光譜。
- **Later / Spike**：塔台語音生成（→ V5 真 ATC）、公開部署 onboarding（束 D，雙裝置 LAN relay 對陌生人的部署門檻）。

## 9. 工法基準（Fable bar — 每個版本 handoff 照交）

- 每版本/階段交付＝一份 handoff doc，含 DoD、test 數、`typecheck/e2e 0 errors`、B1 grep 檢查、必要 trace。
- 全綠才交：vitest 單元 + Playwright e2e（有狀態 server 用 `workers:1`）。
- 安全網是「不可拔的地板」：fallback/heuristic/固定地板；生成內容走三件套（grounded + 結構驗證 + 固定地板 + 逾時）。
- 參數化鐵律（B1）：隨空域/天氣/機場變動的全進 airspace.json，引擎零改動。
- DRAFT 紀律（B5）：生成/草擬內容標 DRAFT 待 Sung 校稿 + 抽查 log。
- Bug fix 走 root-cause + 加「沒修就會紅」的回歸測試。
- HITL 手感閘不可 proxy（鐵律 2）：真機/雙真機由 Sung 親驗，agent 不代簽。
- 每個非顯而易見的決定寫進本檔 §7 拍板紀錄，附「為何」。
- UI 起跑線＝`../spec/DESIGN_CRAFT_GUIDE.md`（TactilePress + 多層表面 + haptic 政策）。
- **素材優先於手刻 ＋ 共存正規化（2026-06-14，見 §11）**：新增 3D 模型前先判類——bespoke 台灣識別件（地標）手刻 voxel（Claude 自足，不需 Blender）；通用件（機種／車／航廈／設備／靶／樹）預設採 CC0 low-poly 現成。**禁止無理由反射性手刻通用模型**（Claude 最燒 loading 的事）；手刻通用件須在 handoff 明示理由（如 trivial 小件比找素材便宜）。**匯入 GLB 一律過正規化**：剝貼圖→flat/Lambert＋重映本款色票＋挑低細節對齊 voxel，否則會打架。既有 voxel 全保留、不遷移；CC0 來源記 attribution（免標也建議列）。
- **套件先行（不只模型，整個 3D 系統，2026-06-14 Sung 拍）**：**任何 3D 工作開動前先問「有沒有現成套件/資產？」，預設用套件、不要反射性 raw THREE 自幹。** 模型→CC0 GLB（`GLTFLoader`）；天空→`three/addons` Sky；霧→內建 `Fog`；雨雪/粒子/特效（含爆炸煙）→粒子套件或 instanced sprite；水→`three/addons` Water；載入/最佳化→three 官方 addon（DRACO/instancing）。**單一 primitive（torus/box/sphere）自幹 OK；成系統的 3D（模型庫/粒子/天氣/地形）一律先 survey 1–2 個套件（授權 CC0/MIT 才用），handoff 寫「為何選套件 X 或為何只能自幹」。** V2 既有 raw-THREE 手做件（voxel 機型/賽道圈/彈丸）已上線不回頭churn；規則對 V3+（天氣最吃）與新資產生效。

## 11. 美術／素材管線 — voxel + low-poly 共存（2026-06-14 拍定）

> **本節取代 §7 之前「voxel 玩具視覺恆定」單一原則**，是 V1→V5 美術方向與 Claude loading 紀律的**單一真相源**；每個新增 3D 內容的 handoff 引用本節。
> **緣起**：技術評估（Sung × Opus，2026-06-14）。論壇工法「不讓 AI 硬畫、改用免費 3D 模組」。本款早已是 Three.js 3D，但美術是**手寫 voxel 方塊座標**（`t34c.js`/`f16.js`/`landmarks.js`）——而手寫座標正是 Claude 最弱、最燒 loading＋最需 HITL 微調的環節（`f16.js` box×24 手調即一例）。後面 ATR-72／B737／A330 ＋ V4 地勤車 ＋ V5 九機場全排隊手刻＝loading 黑洞。
> **關鍵認知**：voxel 與 low-poly **不是二選一、可共存**。兩者同屬「flat-shaded＋無貼圖＋頂點上色」材質家族（voxel＝用方塊的 low-poly），本款 `Lambert + vertexColors` 正是這家族。統一 flat-shaded＋同一色票＋相近細節密度，方塊機體 × 角面客機就是同一個玩具世界。**taipei_pilot 是獨立 3D web 專案，與 Flutter「玩具櫃」平台識別無關、視覺自選。**

### 11.1 方向
- **voxel + low-poly 共存於同一個 flat-shaded 玩具視覺**。不丟 voxel、不全面遷移。
- **通用件 → 優先 CC0 low-poly 現成、不手刻**：機種、地勤車、航廈、機場設備、靶／敵機／泡泡、樹、建築變化。
- **bespoke 台灣識別件 → 保留手刻 voxel**：地標（101／圓山／總統府／中正紀念堂／美麗華／西門紅樓／大安森林…）。CC0 沒有、辨識度＝靈魂，且 **voxel 手刻 Claude 自足、不需 Blender**。
- **既有 voxel 資產（T-34C／F-16／城市／河流／機場）全保留**，不 re-skin、不遷移。
- **素材源**（CC0，鎖**偏角面/低細節**家族最好配——Kenney.nl 最扁平、Quaternius 次之；另 Poly Pizza／Sketchfab CC0）。**套件＝零新增**：`GLTFLoader`/`DRACOLoader`/`InstancedMesh` 都在 `three` 內；資產離線最佳化（選用）才考慮 build-time 的 `gltf-transform`，runtime 不用。

### 11.2 共存的代價＝正規化紀律（不是「丟進去就好」）
匯入的 CC0 GLB 要過一道**正規化**才不打架（打架風險真實但可控）：
- **材質**：剝貼圖/PBR → flat/Lambert（與現有一致）。
- **色票**：重映到本款色票家族（暖、低對比）。
- **細節密度＋scale/朝向**：挑偏方塊/低面數素材；過平滑高面數曲面件擺粗 voxel 旁仍違和，避開。
- **此 spike 已於 2026-06-14 提前跑**（`spike-glb/`，findings：`spike-glb/FINDINGS.md`，截圖 `out/coexist.png`）：Kenney CC0 Car Kit × voxel **共存成立**；材質 Lambert+flat 近乎免費、**色票重映（降飽和/偏暖）是主槓桿**、scale＝量 bbox→公尺逐型別。v4.0 沿用 `normalize()`/`fitToGround()` recipe 正式化（**Sung HITL 給過 2026-06-14**；只剩補色票重映）。`spike-glb/` 保留作 v4.0 種子。

### 11.3 時機 — 共存即可從現在生效（無遷移成本）
- 既然可共存，**不需 V4 大遷移、也不需「V2/V3 先憋 voxel」**。gate 從現在按**逐件成本**判：通用件查 CC0、bespoke 與 trivial 小件（如氣球靶）voxel 手刻仍可能更便宜。
- 真正的量在 **V4（民航＋地勤）/V5（九機場）**＝首次大量 GLB；到時 CC0 low-poly 進場、與既有 voxel 並排即可。

### 11.4 為何（含 lesson learned）
